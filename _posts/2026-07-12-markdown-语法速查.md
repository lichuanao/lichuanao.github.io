## 1. 文章开头（Front Matter）

每篇文章开头必须有这段，用 `---` 包裹：

```yaml
---
title: 文章标题
date: 2026-07-12 12:00:00 +0800
categories: [分类一, 分类二]
tags: [标签一, 标签二, 标签三]
author: lichuanao
pin: false
---
```

- `title`：文章标题，必填
- `date`：发布时间，建议精确到分钟，格式 `YYYY-MM-DD HH:MM:SS +0800`
- `categories`：分类，**只能有一个**，可以是嵌套 `[[外层, 内层]]`（会生成面包屑路径）
- `tags`：标签，**可以有多个**，用逗号分隔
- `author`：作者（不填则用 _config.yml 全局设置）
- `pin: true`：设为 `true` 则在置顶文章中显示，首页最多同时置顶 2 篇

## 2. 标题

```markdown
# 一级标题  ← 页面大标题，建议一篇文章只用一次
## 二级标题  ← 主要章节
### 三级标题  ← 子章节
#### 四级标题  ← 一般文章不推荐用三级以下
```

> Chirpy 主题会自动给 H2-H4 添加左侧锚点，Hover 可以看到 `#` 链接符号。

## 3. 文字格式

```markdown
**加粗**
*斜体*
~~删除线~~
`行内代码`
**_加粗斜体组合_**
```

效果示例：
- **加粗** — 强调
- *斜体* — 轻量强调
- ~~删除线~~ — 已废弃内容
- `行内代码` — 代码或命令

## 4. 链接与图片

```markdown
[访问百度](https://www.baidu.com)

![图片说明](图片URL或本地路径)
  <!-- 本地图片放在 assets/img/ 下 -->
  <!-- 例如：assets/img/my-screenshot.png -->
```

> 图片建议优先用本地路径（方便离线预览），推送后 GitHub 会自动构建到网站。

```markdown

- 第二项
```

**有序列表：**

## 7. 代码块

**行内代码**：`` `代码` ``

**多行代码块**，在 ``` 后面指定语言：

````markdown
```python
def hello(name):
    print(f"Hello, {name}!")
```
````

常用语言标记：

| 语言 | 标记 | 说明 |
|---|---|---|
| Python | `python` | 脚本 |
| Bash/Shell | `bash` / `shell` | 命令行 |
| PowerShell | `powershell` | Windows 命令 |
| JavaScript | `javascript` / `js` | 前端脚本 |
| JSON | `json` | 配置文件 |
| YAML | `yaml` | 配置文件 |
| Markdown | `markdown` | 语法演示 |
| CSS | `css` | 样式 |
| HTML | `html` | 网页 |
| SQL | `sql` | 数据库 |
| Go | `go` | Go 语言 |
| C / C++ | `c` / `cpp` | C 系列 |

## 11. ECharts 图表（Chirpy 特有）

## 16. 常见写作注意事项

### ✅ 推荐的写法
- 标题简洁，一眼能看出文章主题
- 段落不要太长，适当空行分段
- 代码块注明语言，方便阅读
- 图片加说明文字（alt text）

### ⚠️ 容易出错的地方
- `date` 格式必须是 `YYYY-MM-DD HH:MM:SS +0800`，否则 Jekyll 构建可能报错
- 文件名格式必须是 `YYYY-MM-DD-标题.md`，否则不会被识别为文章
- `_posts/` 目录下的文件名日期要跟 `date` 字段一致
- 中文引号 `" "` 和英文引号 `" "` 不要混用，推荐全部用英文

### ❌ Chirpy 不支持的功能
- Markdown 里的 HTML `<script>` 标签（出于安全考虑会被过滤）
- 多行 LaTeX 表格（用图片代替）
- HTML 表格的复杂合并单元格（建议用 Markdown 原生表格）
